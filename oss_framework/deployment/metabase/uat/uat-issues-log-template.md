## Issue Severity Definitions

| Severity | Definition | Examples | Response Time |
|----------|------------|----------|---------------|
| **Critical** | System is unusable or major security breach | Login fails for all users; Student names visible to board members; Database corruption | Immediate (< 2 hours) |
| **High** | Major functionality broken, no workaround | Dashboard won't load; Export crashes Metabase; Wrong data displayed | Same day (< 8 hours) |
| **Medium** | Functionality impaired but workaround exists | Filter doesn't work but manual SQL does; Slow load times; Minor data inaccuracy | Within 3 days |
| **Low** | Cosmetic or minor inconvenience | Chart colors are hard to read; Typo in label; Tooltip missing | Before production launch |
